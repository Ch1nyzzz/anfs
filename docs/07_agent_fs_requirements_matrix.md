# Agent Filesystem Requirements Matrix

Source: Sarah Catanzaro, "File systems for agents", Amplify Partners,
May 27, 2026.
https://www.amplifypartners.com/blog-posts/file-systems-for-agents

The article argues that agents naturally operate on file-like data, but future
agent workloads need stronger concurrency, query, materialization, access
control, and scale support than traditional filesystems provide. This matrix
maps those requirements to ANFS.

## Current Priority

ANFS should now focus on making the kernel smaller and harder:

1. Keep canonical state and invariants explicit in `06_kernel_contract.md`.
2. Use `09_complexity_audit.md` to reject redundant canonical state before it
   enters the kernel.
3. Centralize visibility policy so every projection enforces labels the same
   way.
4. Harden ordinary filesystem compatibility through cache hit-rate benchmarks,
   contention tests, and FUSE evaluation.
5. Publish official real LoCoMo snapshot runs and larger filesystem workflow
   snapshots.

The current executable conformance proof is documented in
`08_agent_native_fs_conformance.md` and implemented in
`tests/test_conformance.py::test_agent_native_fs_conformance_preserves_kernel_invariants_with_existing_tools`.
